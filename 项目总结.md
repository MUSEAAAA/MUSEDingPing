# 黑马点评 (hm-dianping) — 项目总结

## 项目概述

本地生活服务平台后端，类似大众点评/美团，支持用户浏览店铺、探店笔记、点赞互动、领取秒杀优惠券。核心亮点聚焦在 **Redis 在高并发业务场景中的高阶应用**。

---

## 核心业务模块

### 1. 用户模块（UserServiceImpl）

- **手机号验证码登录**：验证码存 Redis `login:code:{phone}`，2 分钟过期
- **Token 登录**：登录后生成随机 token，用户信息以 **Redis Hash** 存储（`login:token:{token}`），10 小时过期
- **双拦截器机制**（MvcConfig.java）：
  - `RefreshTokenInterceptor`（order=0，拦截 `/**`）：解析 token 将用户恢复至 ThreadLocal，并**自动续期 token 过期时间**
  - `LoginInterceptor`（order=1，拦截除白名单外路径）：检查 `UserHolder` 是否为空，为空则返回 401
  - 实现**无状态登录**，避免传统 Session 在分布式环境下的共享问题

### 2. 商铺模块（ShopServiceImpl + CacheClient）

**Redis 缓存三大问题的系统性解决**：

- **缓存穿透**（`queryWithPassThrough`）：查不到数据时，缓存空值 `""`（2 分钟过期），防止恶意请求穿透到 DB
- **缓存击穿**（`queryWithLogicalExpire`）：
  - 使用**逻辑过期策略**：Redis 中存储 `RedisData` 对象（数据 + 逻辑过期时间）
  - 请求发现数据"已过期" → 获取互斥锁（`setIfAbsent`）→ 启动独立线程重建缓存 → 旧线程直接返回旧数据
  - 缓存重建使用**10 线程独立线程池** `CACHE_REBUILD_EXECUTOR`
- **缓存双写一致性**：更新店铺采用**先更新数据库，再删除缓存**策略，配合 `@Transactional` 事务保证原子性

### 3. 秒杀模块（VoucherOrderServiceImpl）— **核心亮点**

高并发秒杀的全链路设计，经历了多轮方案演进（代码中保留注释）：

```
纯 DB → synchronized + 事务 → SimpleRedisLock → Lua + Redis Stream 异步下单
```

**最终实现**：

- **Lua 脚本原子限流**（`seckill.lua`）：一次 Redis 调用完成库存扣减 + 一人一单判断，返回 0/1/2
  ```lua
  -- 核心逻辑
  local stock = redis.call('get', stockKey)
  if stock <= 0 then return 1 end
  if redis.call('sismember', orderKey, userId) == 1 then return 2 end
  redis.call('incrby', stockKey, -1)
  redis.call('sadd', orderKey, userId)
  redis.call('xadd', 'stream.orders', '*', ...)
  ```

- **Redis Stream 异步削峰**：秒杀请求只操作内存级的 Redis，订单落库由消费者线程异步处理
- **容错机制**：正常消费 Stream 新消息；异常时读取 `pending-list` 兜底处理，确保消息不丢失
- **Redisson 可重入锁**：异步下单时保证一人一单的事务安全
- **事务代理**：通过 `AopContext.currentProxy()` 获取代理对象，解决 `@Transactional` 方法内部调用失效问题

### 4. 全局 ID 生成器（RedisWorker）

- 利用 Redis INCR 按天生成自增序列（`icr:order:yyyy:MM:dd`）
- 最终 ID = **时间戳偏移（高 32 位）| 当天自增序列（低 32 位）**
- 保证全局唯一、趋势递增，比 UUID 节省空间，适合分库分表场景

### 5. 分布式锁（两种实现）

| 实现 | 特点 |
|------|------|
| **SimpleRedisLock（自制）** | 基于 `SETNX` + Lua 脚本保障判断与释放的原子性，支持线程标识防误删 |
| **RedissonClient（第三方）** | 可重入锁，自带 WatchDog 看门狗自动续期 |

### 6. 探店笔记模块（BlogServiceImpl）

- **点赞**：基于 Redis Set（`blog:liked:{blogId}`）保证唯一点赞，`isMember` 判断状态
- **热门笔记**：按点赞数降序分页查询
- **笔记详情**：附带作者姓名、头像以及当前用户点赞状态

### 7. 店铺分类模块（ShopTypeServiceImpl）

- 分类列表缓存至 Redis（`cache:shopType:list`）
- 缓存未命中则回查 MySQL 并写入缓存

### 8. 秒杀券管理（VoucherServiceImpl）

- 创建秒杀券时同步写入 MySQL（`tb_voucher` + `tb_seckill_voucher`）并将库存同步至 Redis（`seckill:stock:{id}`）
- 优惠券列表通过 LEFT JOIN 关联查询普通券与秒杀券

### 9. 图片上传（UploadController）

- UUID 重命名文件，按 hash 值散列至两级目录，上传至 nginx 静态目录

### 10. 全局异常处理（WebExceptionAdvice）

- `@RestControllerAdvice` 统一拦截 `RuntimeException`，返回统一 JSON 响应

### 11. 密码加密（PasswordEncoder）

- **加盐 MD5**：随机 20 位盐，格式 `{salt}@{md5(password+salt)}`

## 面试可重点阐述的技术亮点

1. **Redis 缓存三大问题的系统性解决**：穿透（空值缓存）、击穿（逻辑过期 + 互斥锁异步重建）、雪崩（差异化 TTL）
2. **秒杀系统的全链路优化**：从纯数据库 → Lua 脚本预扣库存 → Redis Stream 异步落库，热路径与冷路径完全分离
3. **分布式锁方案演进**：从 `synchronized` → `SETNX` 自制锁（Lua 脚本） → Redisson，体现对原理的理解和工程化取舍
4. **双拦截器 Token 自动续期**：无状态登录方案，适合分布式水平扩展
5. **全局 ID 生成器**：时间戳 + Redis 自增，兼顾唯一性与有序性，预适配分库分表
6. **代码演进痕迹**：代码注释中保留了从简单方案到高并发方案的迭代过程，面试时可展示"秒杀做了三个版本"的优化思路
