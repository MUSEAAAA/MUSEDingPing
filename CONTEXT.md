# 秒杀优惠券平台 — 通用语言

## 核心领域

| 术语 | 定义 |
|------|------|
| 秒杀券 (SeckillVoucher) | 限时限量的优惠券，有库存、起止时间，与普通券一对一关联 |
| 库存 (Stock) | 秒杀券可被领取的数量。预热到 Redis，扣减由 Lua 脚本原子执行 |
| 一人一单 | 同一用户对同一秒杀券只能下一单的约束 |
| 异步下单 | 秒杀请求在 Redis 中判定资格后，通过 Redis Stream 异步写 MySQL，热路径与冷路径分离 |

## 防刷领域

| 术语 | 定义 |
|------|------|
| IP 限流 (Rate Limiting) | 同一 IP 在固定时间窗口内请求次数超过阈值时拒绝服务 |
| 固定窗口 (Fixed Window) | 以自然秒为单位的计数窗口，key 含秒级时间戳，INCR + EXPIRE 实现 |
| 黑名单 (Blacklist) | 累计被限流次数超过阈值的 IP 集合，设有时效，过期自动解封 |
| 风控日志 (Risk Log) | 记录限流/拉黑/解封事件的审计轨迹 |

## 缓存领域

| 术语 | 定义 |
|------|------|
| 缓存穿透 | 查询不存在的数据，请求穿过缓存直达 DB。对策：空值缓存 |
| 缓存击穿 | 热点 key 过期瞬间大量请求涌向 DB。对策：逻辑过期 + 互斥锁异步重建 |
| 缓存雪崩 | 大量 key 同时过期导致 DB 过载。对策：TTL 加随机偏移 |

## 分布式锁领域

| 术语 | 定义 |
|------|------|
| SETNX 锁 | 基于 Redis SETNX 的自制锁，Lua 脚本保证释放原子性，线程标识防误删 |
| Redisson 锁 | 生产级可重入分布式锁，WatchDog 自动续期 |
